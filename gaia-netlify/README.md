# GAIA — 脱炭素ホログラムエージェント

タブレット + ピラミッドシートで投影できる、脱炭素特化のAIエージェント。
Gemini APIをサーバーサイドで呼び出すため、APIキーが漏洩しない構成です。

---

## ファイル構成

```
gaia-netlify/
├── netlify.toml              # Netlify設定（リダイレクト等）
├── package.json              # Node.js設定
├── public/
│   └── index.html            # フロントエンド（3D描画、音声I/O）
└── netlify/
    └── functions/
        └── gemini.js         # サーバーサイドAPIプロキシ
```

## デプロイ手順（最も簡単な方法：ドラッグ&ドロップ）

### ステップ 1: APIキーを準備

1. https://aistudio.google.com/apikey にアクセス
2. **これまでチャットに貼ったキーがあれば、必ず削除**
3. 「Create API key」で新しいキーを発行
4. キー（`AIza...`）をコピー（**チャット等には貼らない**）

### ステップ 2: Netlifyにデプロイ

1. https://app.netlify.com にログイン（GitHubアカウントなどで無料登録可）
2. ダッシュボードで「**Add new site**」→「**Deploy manually**」
3. このプロジェクトのフォルダ（`gaia-netlify` 全体）をドラッグ&ドロップ
4. デプロイが開始されます（30秒〜1分）

### ステップ 3: 環境変数を設定（最重要）

1. デプロイ後、サイトの管理画面で「**Site configuration**」→「**Environment variables**」を開く
2. 「**Add a variable**」→「**Add a single variable**」
3. 以下を入力：
   - **Key**: `GEMINI_API_KEY`
   - **Value**: ステップ1でコピーした新しいAPIキー
   - **Scopes**: 全部チェック（または "All scopes"）
4. 「**Create variable**」で保存

### ステップ 4: 再デプロイ

環境変数を反映するため、再デプロイが必要です。

1. 「**Deploys**」タブを開く
2. 最新のデプロイで「**Trigger deploy**」→「**Deploy site**」
3. 完了したら、サイトURL（例: `https://your-site-name.netlify.app`）にアクセス

タブレットのブラウザでURLを開けば、すぐに使えます。

## 機能

### 🎤 マイクの常時ON/OFFモード

画面右上の「MIC OFF / MIC ON」ボタンをタップすると、マイクが常時ONになります。
ONの間は、話しかけると自動で音声を拾い、1.2秒の沈黙でメッセージとして送信されます。
ハンズフリーで対話できます。

### 🔊 音声読み上げON/OFF

下部の「🔊 OFF / ON」ボタンで、GAIAの返答を日本語で読み上げる機能を切り替えできます。

### 🛑 GAIAの発話を中断する（バージイン）

GAIAが話している途中でも、いつでも中断できます。3つの方法があります。

1. **声で割り込む（マイクモードON時）**: GAIAが話している最中に何か話し始めると、自動的にGAIAが黙ってあなたの声を聞き始めます。自然な会話の流れで割り込めます。
2. **音量で割り込む（マイクモードON時）**: 大きな音や声を出すだけで瞬時にGAIAが停止します。「あ、」と一言言うだけで止まります。話し始める前に手動で中断したい時に便利です。
3. **STOPボタン**: GAIAが話している間、画面中央下に赤い「STOP」ボタンが表示されます。タップで即座に停止。マイクOFFでも使えます。

GAIA発話中、画面下部に細い色付きバーが表示されます。これは現在のマイク入力レベルです。一定以上に達するとバージインが発動します。

**エコー対策**: GAIA自身の音声を「あなたの声」と誤認しないよう、発話開始から0.6秒間は中断検知を無効にし、ブラウザのエコーキャンセル機能も有効にしています。

### 🎙 音声を選ぶ

🔊ボタンの隣の🎙ボタンで、利用可能な日本語音声の一覧が開きます。各音声を「▶ 試聴」で確認し、最も自然に聞こえるものを選択してください。「ローカル」と表示される音声は端末内蔵で品質が安定しています。

**自然な音声を増やすには:**
- **Windows**: 設定 → 時刻と言語 → 音声 → 音声の管理 → 追加 → 「日本語」を選択
- **Mac**: システム設定 → アクセシビリティ → 読み上げコンテンツ → システムの声 → 「Kyoko」「Otoya」をダウンロード
- **iOS/iPadOS**: 設定 → アクセシビリティ → 読み上げコンテンツ → 声 → 日本語 → 「拡張」または「プレミアム」をダウンロード（最も自然）
- **Android**: 設定 → ユーザー補助 → テキスト読み上げ → Google音声合成エンジンを使用

**特におすすめの自然な日本語音声:**
- iOS: 「Kyoko (拡張)」「O-Ren」
- Mac: 「Kyoko (拡張)」
- Chrome (任意のOS): 「Google 日本語」
- Android: 「Google 日本語」

### 対話モード / 投影モード

- **対話モード**: 通常のチャット画面。3Dエージェントとテキスト/音声で対話。
- **投影モード**: 4分割画面。タブレットを水平に置き、ピラミッドシートを中央に置くと、立体ホログラムとして見えます。

### 3Dエージェントの構成

- 緑のシェーダーで揺らぐコア
- 2層のワイヤーフレーム外殻
- 周回するCO₂粒子（捕捉されるイメージ）
- 軌道リング
- GAIAが話すと: 放射状の波動パルス、金色のオーラ、粒子の高速化

## セキュリティ

- APIキーはNetlifyのサーバー環境変数にのみ保存され、ブラウザには一切送信されません
- ブラウザは `/api/gemini` という自サイトのエンドポイントを叩くだけです
- Gitリポジトリにキーを含めず、`.env`ファイルもコミットしないでください

## 注意点

### マイクが動作しない場合

- HTTPSでアクセスしているか確認（Netlifyは自動でHTTPS対応）
- ブラウザでマイク許可を求められたら「許可」を選択
- Chrome/Edge/Safari（最新）推奨。一部Androidの古いブラウザは非対応

**Android Chromeで「赤くパルスするのに認識しない」場合**

画面左下の「DEBUG」ボタンをタップしてログを表示し、エラー内容を確認できます。よくある原因と対処法:

1. **`network`エラー** - Google音声検索サービスに接続できない
   - 設定 → アプリ → Google → 有効化
   - 設定 → 言語と入力 → 仮想キーボード → 「Google音声入力」が有効か確認
   - 機内モード等でネット切断されていないか確認

2. **音声認識が勝手に止まる** - Android Chromeの仕様で、`continuous: true`を設定しても5〜10秒で自動停止します。本アプリはAndroid向けに自動再起動とウォッチドッグを実装済みです。

3. **画面ロックで止まる** - Androidは画面オフでマイクを切るため、対話中は画面オン推奨

4. **他アプリがマイクを占有** - Google Assistant、Zoom等を終了してから試す

**iOSで音声認識が動かない**

iOS（iPhone/iPad）のSafariおよびChromeは、`webkitSpeechRecognition`を実装していません。これは仕様であり、回避できません。
- 代替: iOS標準のキーボード音声入力（テキスト欄をタップ→マイクアイコン）を使う

### 音声読み上げがOSによって違う

- iOS: Kyokoなど標準音声
- Android: Google日本語TTS
- Mac: Kyoko / Otoya
- Windows: Haruka / Sayaka

### 無料枠について

- Netlify: 月100GB帯域、月300関数実行時間まで無料
- Gemini: 1日あたり数百〜数千リクエスト無料（モデルによる）

個人利用なら通常無料枠で収まります。

## ローカル開発（オプション）

Netlify CLIを使えばローカルで試せます：

```bash
npm install -g netlify-cli
cd gaia-netlify
echo "GEMINI_API_KEY=AIza..." > .env
netlify dev
```

`http://localhost:8888` で動作確認できます。

## トラブルシューティング

### 「エラー (500): GEMINI_API_KEY is not set on the server」
→ Netlifyの環境変数が設定されていない、または再デプロイされていません。ステップ3〜4を再確認してください。

### 「エラー (429): Quota exceeded」
→ Geminiの無料枠を超過。少し待つか、Cloud Consoleで課金設定を確認。

### 画面が真っ黒
→ JavaScriptエラーの可能性。F12（開発者ツール）でコンソールを確認してください。
